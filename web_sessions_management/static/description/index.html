Manage Users Login Rules in Odoo

Access Rules by Group or User and by Company

Group/User Login Configuration

Define login rules by groups or single users, company dependent in a multicompany configuration.

  1. Allow or not multiple sign in of same user; [DONE]
  2. Define session timeouts; [DONE]
  3. Define a time/week calendar by day where users can login; [DONE]
  4. You can have user exceptions, overwriting group settings in user settings. The most restrict rule will be applied. [DONE]

Global Sessions Management

Better Security!

Administrators can manage easily the open sessions, and have control of who is logged in, giving better view of users behaviours, improving overall security and control.
Access it at Settings>Technical>Security menu.

  1. Global sessions logs; [DONE]
  2. Graph view with Group By: [DONE]
    • Session state;
    • Login/Logout date time;
    • User;
    • and others...
  3. Close any active session. [DONE]
  4. Close all active sessions. [TODO]
  5. Send email to users with active sessions. [TODO]

NOTE: admin user has no access restriction.

Self Session Management

Full Control of Your Own Sessions

User Settings Sessions Control

Every user can manage his own sessions from preferences, improving overall security:

  1. Users can see their own log rules to login. The most restrictive between group and user ones will be applied.
    (eg:. if a group defines 4 hours, and user define 30 minutes the session will expires in 30 minutes, the earlier); [DONE]
  2. Users can close related active session; [TODO]
